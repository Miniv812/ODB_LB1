# Лабораторна робота 1: Збір вимог та розробка схеми ER
Наша задача розробити БД для гри (з можливістю розширення).

За сутность візьмемо основне, що є в грі, це: Персонаж, Гільдія, Предмети, Зони, Квести та Чудовиська. 

Для створення персонажа нам потрібна реальна людина, тому додаємо ще сутність Юзера. 

Придумаємо атрибути для цих сутностей та певні зв'язки між ними та проектуємо усі наші рішення на єкран.

Зв'язки:

1)Гравець може мати кілька персонажів, а персонаж має тальки одного гравця

2)Гільдія може мати кілька персонажів, а персонаж може мати тільки одну гільдію

3)Персонаж може мати кілька предметів, а предмет може мати кілька персонажів

4)Гільдія може мати або не мати одну локацію, а локація може мати чи не мати одну гільдію

5)Локація може мати багато существ, а существа можуть мати багато локацій

6)Существа можуть мати предмети, а можуть мати существ

7)Существа можуть мати квести, а квести можуть мати существ через вбивство

8)Квести можуть буду в різних локаціях і в локація кілька квестів

9)Квести можуть бути в гільдії, а в гільдії можуть бути кілька квестів

10)Квести можуть давати предмети, а предмет може бути тільки в одному квесті
